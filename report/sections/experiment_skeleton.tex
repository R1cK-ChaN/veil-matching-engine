\section{Verification Results}

\subsection{Purpose}

The experiments in this project are verification experiments rather than
systems-performance experiments. Their role is to validate the specification,
surface counterexamples early, and document the scope of the checks used
during development.

\subsection{Deductive Invariant Checking}

The main result comes from Veil's deductive invariant checking. The current
model declares seven safety properties and thirteen supporting invariants.
Together with the generated \texttt{doesNotThrow} obligation, this gives
twenty-one checks per verification block. Veil checks initialisation and each
of the three actions (\texttt{doMatch}, \texttt{submit\_sell}, and
\texttt{submit\_buy}), producing eighty-four preservation checks in total. All
eighty-four pass.

\subsection{Bounded Model Checking (Disabled)}

We initially also planned to invoke Veil's \texttt{\#model\_check} as a small
finite smoke test over two traders and three order identifiers. In practice
this command requires a \texttt{Veil.Enumeration} instance for each action's
argument tuple, and both submission actions take \texttt{Nat}-typed price and
quantity arguments. \texttt{Nat} has no finite enumeration, so the synthesised
\texttt{instEnumerationLabel} reduces to \texttt{sorry} and the bounded
checker cannot evaluate. Re-enabling the check would require restricting price
and quantity to a finite type such as \texttt{Fin~N}, which would narrow the
abstract model. We chose to keep the unbounded specification, since the
deductive result already covers it.

\subsection{Interpretation}

The deductive checks establish that the declared safety and invariant set is
preserved by initialisation and all three actions in the unbounded model.
That, rather than a bounded smoke test, is the primary verification result
reported in this paper.

\subsection{Verification Time}

Wall-clock numbers on a Linux laptop with no special hardware: a cold
\texttt{lake build MatchingEngine} that has the dependency caches already
populated takes roughly nine minutes for the SMT-backed checks to clear; an
incremental \texttt{lake build} with a warm cache and no source change
completes in about one second. A first-time clean checkout, which has to
build mathlib, cvc5, and Veil, takes roughly fifteen to thirty minutes
depending on cache state.
