\section{Verification Results}

\subsection{Purpose}

The experiments in this project are verification experiments rather than
systems-performance experiments. Their role is to validate the specification,
surface counterexamples early, and document the cost and scope of the bounded
checks used during development.

\subsection{Deductive Invariant Checking}

The main result comes from Veil's deductive invariant checking. In the current
model there are seven declared safety properties and thirteen supporting
invariants. Together with the generated \texttt{doesNotThrow} obligation, this
gives twenty-one checks per verification block. Veil checks initialization and
each of the three actions, namely \texttt{doMatch}, \texttt{submit\_sell}, and
\texttt{submit\_buy}. The current run therefore covers eighty-four
initialization-and-action preservation checks, and all of them pass.

\begin{itemize}[leftmargin=1.5em]
\item \textbf{Safety properties:} 7
\item \textbf{Supporting invariants:} 13
\item \textbf{Verification blocks:} initialization, \texttt{doMatch},
\texttt{submit\_sell}, \texttt{submit\_buy}
\item \textbf{Total deductive checks:} 84
\item \textbf{Result:} all checks pass
\end{itemize}

\subsection{Bounded Model Checking}

The bounded experiment is only a smoke test over a tiny finite instance. It
does not sweep bounded price and quantity domains. Instead, it fixes a small
trader universe and a small order-ID universe and asks Veil to search for a
violation of the declared safety and invariant set.

\begin{itemize}[leftmargin=1.5em]
\item \textbf{Finite bound:} \texttt{trader := Fin 2},
\texttt{orderId := Fin 3}
\item \textbf{Theory values:} constant initial balances for cash and assets
\item \textbf{Observed result:} no violation found
\item \textbf{Explored states:} 1
\end{itemize}

\subsection{Interpretation}

These results support the current paper claims, but only at the scale of the
abstract legal-match model implemented here. The deductive checks establish
that the declared safety and invariant set is preserved by initialization and
the three actions. The bounded model check is useful as a sanity check, but
its finite scope is too small to support stronger experimental claims about
larger bounded price and quantity domains.
