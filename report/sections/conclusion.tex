\section{Conclusion}

This project formalises a simplified price-time-priority matching engine as a
transition system in Veil and verifies seven safety properties covering
best-order selection, trade-field provenance, no self-dealing, and a
frame-style balance conservation property. Thirteen supporting invariants make
the safety properties inductive, and all eighty-four deductive checks pass.
Even in this narrow single-asset, limit-order-only setting, the model combines
ordered-data reasoning, trade-log provenance, and accounting invariants in a
way that exercises Veil's verification infrastructure meaningfully.
