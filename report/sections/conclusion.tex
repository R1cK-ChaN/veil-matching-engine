\section{Conclusion}

This project treats a simplified price-time-priority matching engine as a
transition-system verification problem. Even in a narrow single-asset,
limit-order-only form, the model already combines ordered-data reasoning,
trade-log provenance, and accounting invariants in a way that makes the proof
interesting. A successful outcome is a clean Veil specification, a core set of
checked safety theorems, supporting Lean lemmas, and a small but convincing
verification experiment suite.
